***VPC: Virtual Private Cloud***:<br>
→ Vpc is virtual isolated network creating in the Cloud<br>
→ By Default, AWS will provide a default vpc in every region at the time of aws account creation.<br>
→ As Per Requirement, we can create VPC in every region and we call those VPCs are Custom VPC.<br>
→ In Virtual Isolated Network  you can launch resources and connect to the resources privately.<br>

***1. Subnet***
```
Subnets divide VPC into smaller networks.
2 Types : Public Subnet and Privte Subnet.
1. Public Subnet : Has route to Internet Gateway
   Used for: Web servers , Bastion host and Load Balancer
2. Private Subnet : No direct internet access
   Used for: Database and Backend servers
```
***2. Internet Gateway (IGW)***
```
Allows communication between VPC and internet.
Without IGW → No internet access.
Internet Gatway is a gateway which can make any subnet as a Public subnet.
Note: we can attach only 1 internet gateway to any vpc
```
***3.Route Table***
```
Route Table Will contains the routes which are required to establish communication for the subnets.
Route: Provide one path to reach the destination.
AWS will provide a default Route Table Called “Main Route Table”
Main Route Table will act as a Private Route Table.
When You attach IGW in the  Public RT With the Subnets become Public Subnets.

VPC Sizing : In Amazon Web Services, you must assign a CIDR block when creating a VPC.
. VPC Size : Example: 10.0.0.0/16 , /16 = 65,536 IP addresses and AWS allows VPC sizes from /16 to /28
. Subnet Size : Example: /24 , /24 = 256 IPs and AWS reserves 5 IPs → 251 usable IPs
. Private IP Ranges (RFC1918) : 10.0.0.0/8 → 16M IPs, 172.16.0.0/12 → 1M IPs and 192.168.0.0/16 → 65K IPs
. IP Types : Private IP → Internal communication, Public IP → Internet access and Elastic IP → Static public IP
```

***FAQ***</br>
***1. How to identify whether a Subnet is Public or Private?***
```
Check the Route Table associated with the subnet:
Public Subnet : Has a route like: 0.0.0.0/0 → Internet Gateway (IGW) and Instances can access the internet directly (if they have public IP)
Private Subnet : No direct route to IGW and Usually has: 0.0.0.0/0 → NAT Gateway and Used for backend resources (DB, app servers)
Shortcut: IGW → Public and NAT / No IGW → Private
```
***2. Can we attach multiple CIDR blocks to a VPC?***
```
Yes, you can.
When IPs are exhausted, you can add secondary CIDR blocks to the VPC
After adding, you can create new subnets from the new CIDR range
Example: Primary CIDR: 10.0.0.0/16 and Add secondary CIDR: 192.168.0.0/16
Useful when scaling large applications
```
***3. What is VPC and its Components?***
```
VPC (Virtual Private Cloud)
A logically isolated network in AWS where you launch resources like EC2.
Main Components of VPC:
1. CIDR Block : Defines IP range (e.g., /16, /24)
2. Subnets : Divide VPC into smaller networks (Public / Private)
3. Route Tables Control traffic routing
4. Internet Gateway (IGW) : Enables internet access
5. NAT Gateway : Allows private subnet to access internet (outbound only)
6. Security Groups : Instance-level firewall
7. Network ACL (NACL) : Subnet-level firewall
8. Elastic IP : Static public IP
9. VPC Peering / VPN / Direct Connect : Connectivity options
```
