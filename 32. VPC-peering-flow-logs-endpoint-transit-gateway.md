***VPC -Peering***<br>
VPC Peering is a connection between two VPCs that allows them to communicate privately using their private IP addresses.<br>

Types of VPC Peering
```
1. Same Region (Most Common) : Both VPCs in same region (e.g., Mumbai) and Easy setup
2. Cross Region (Inter-Region Peering) : VPCs in different regions
   Example: VPC-1 → ap-south-1 and VPC-2 → us-east-1
3. Cross Account : Different AWS accounts and Requires manual acceptance
```
Task: VPC Peering (Cross-Region) VPC-1: 192.168.0.0/16 → Northern Virginia and VPC-2: 172.31.0.0/16 → Mumbai

```
Step 1: Create VPC-1 (N. Virginia)
Switch region → N. Virginia (us-east-1)
Go to VPC → Create VPC
Name: VPC-1 and CIDR: 192.168.0.0/16
```

```
Step 2: Create VPC-2 (Mumbai)
Switch region → Mumbai (ap-south-1)
Go to VPC → Create VPC
Name: VPC-2 and CIDR: 172.31.0.0/16
```
<img width="1911" height="837" alt="image" src="https://github.com/user-attachments/assets/b5b4f8fc-ef27-447d-9448-78dd20aace21" />

```
Step 3: Create VPC Peering Connection
. Go to VPC → Peering Connections → Create
. Name: VPC1-VPC2-Peering
. Requester VPC: VPC-1 (N. Virginia)
. Accepter VPC: Account: same (or different) and Region: Mumbai VPC: VPC-2
. Click Create
```
<img width="1881" height="745" alt="image" src="https://github.com/user-attachments/assets/35e0729f-49fa-4763-9348-fa643a63830f" />

```
Step 4: Accept Peering (Mumbai Region)
. Switch to Mumbai region , Go to Peering Connections
. Select request → Accept Request
. Status → Active
```
<img width="1912" height="742" alt="image" src="https://github.com/user-attachments/assets/92b67495-ef6b-4d78-95c9-2789f91239b3" />

```
Step 5: Update Route Tables (VERY IMPORTANT)
. In VPC-1 (N. Virginia)
. Go to Route Tables
. Add route: Destination: 172.31.0.0/16 → Target: Peering Connection
```

```
In VPC-2 (Mumbai)
. Go to Route Tables
. Add route: Destination: 192.168.0.0/16 → Target: Peering Connection
```
