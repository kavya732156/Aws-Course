***NAT: Network Address Translation***<br>
1. NAT service is used to share the internet with the Private subnet
2. NAT acts as a mediator between the private subnet & the internet, it will redirect all the traffic to the interent to enable internet browsing.
3. NAT will never allow incoming traffic to the private subnet instances.
4. NAT service must be created in the PUBLIC SUBNET only.
5. After Installing NAT Service in the Public Subnet. Modify the Private route Table Routes.<br>

***How NAT works?***<br>
```
. Private instance sends request to internet
. NAT Gateway replaces private IP → public IP
. Request goes to internet
. Response comes back to NAT
. NAT sends it back to the private instance
```

***Types of NAT in AWS***
```
1. NAT Gateway (Recommended) : Managed by AWS, Highly available , Placed in public subnet and Uses Elastic IP
   NAT Gateway is Not free-Tier Eligible, charged on hourly basis.
2. NAT Instance (Old method) : EC2 instance configured as NAT and Needs manual setup & maintenance
```

***NAT Instance:***
A NAT Instance is an EC2 instance configured to act as a NAT device, allowing private subnet instances to access the internet.<br>
How NAT Instance Works?
```
. Private EC2 sends request (e.g., to install packages)
. Traffic goes to NAT Instance
. NAT Instance replaces private IP → public IP
. Sends request to internet via Internet Gateway
. Response comes back → NAT Instance → Private EC2
```
***Steps to Create NAT Instance***
```
. Launch EC2 in Public Subnet
. Assign Elastic IP
. Disable Source/Destination Check
. Configure IP forwarding: sudo sysctl -w net.ipv4.ip_forward=1
. Configure iptables: sudo iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
. Update Private Route Table → point to NAT Instance
```
***NAT Instance vs NAT Gateway***
```
| Feature      | NAT Instance            | NAT Gateway      |
| ------------ | ----------------------- | ---------------- |
| Management   | Manual                  | AWS Managed      |
| Availability | Single point of failure | Highly available |
| Scaling      | Manual                  | Auto scaling     |
| Cost         | Cheaper                 | Slightly higher  |
| Performance  | Limited by EC2 size     | High performance |
```
