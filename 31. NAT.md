***NAT: Network Address Translation***<br>
1. NAT service is used to share the internet with the Private subnet
2. NAT acts as a mediator between the private subnet & the internet, it will redirect all the traffic to the interent to enable internet browsing.
3. NAT will never allow incoming traffic to the private subnet instances.
4. NAT service must be created in the PUBLIC SUBNET only.
5. After Installing NAT Service in the Public Subnet. Modify the Private route Table Routes.<br>

***How NAT works?***<br>
```
. Private instance sends request to internet
. NAT Gateway replaces private IP → public IP
. Request goes to internet
. Response comes back to NAT
. NAT sends it back to the private instance
```

***Types of NAT in AWS***
```
1. NAT Gateway (Recommended) : Managed by AWS, Highly available , Placed in public subnet and Uses Elastic IP
   NAT Gateway is Not free-Tier Eligible, charged on hourly basis.
2. NAT Instance (Old method) : EC2 instance configured as NAT and Needs manual setup & maintenance
```
***Exercise 1: Create a VPC with Public and Private Subnets***<br>
```
Step 1: Create VPC
Go to VPC Dashboard → Create VPC values: Name: MyVPC and CIDR Block: 10.0.0.0/16
This gives ~65,536 IPs
<img width="1882" height="727" alt="image" src="https://github.com/user-attachments/assets/8f6b081d-0275-4c13-a679-8b17c83b0734" />

Step 2: Create Subnets
. Public Subnet Name: Public-Subnet , CIDR: 10.0.1.0/24 and Availability Zone: Any (e.g., ap-south-1a)
. Private Subnet : Name: Private-Subnet, CIDR: 10.0.2.0/24 and Availability Zone: Same or different AZ

Step 3: Create Internet Gateway (IGW)
. Go to Internet Gateways → Create : Name: MyIGW and Attach it to MyVPC

Step 4: Create Route Table for Public Subnet
. Create Route Table → Name: Public-RT
. Attach to VPC
. Add Route: Destination: 0.0.0.0/0 and Target: Internet Gateway (IGW)
. Associate: Attach this route table to Public Subnet
Now this subnet becomes Public

Step 5: Create NAT Gateway
Go to NAT Gateways → Create
Select: Subnet: Public Subnet and Allocate Elastic IP

Step 6: Create Route Table for Private Subnet
. Create Route Table → Name: Private-RT
. Add Route: Destination: 0.0.0.0/0 and Target: NAT Gateway
. Associate: Attach to Private Subnet
Now this subnet becomes Private with internet access (outbound only)

Step 7: Launch EC2 Instances
. Public Instance:  Launch in Public Subnet and Enable Auto-assign Public IP
. Private Instance : Launch in Private Subnet and No public IP

Step 8: Testing
. connect into Public EC2 → ping goole.com it works
. From Public EC2 → connect into Private EC2 (via private IP)
. From Private EC2: ping google.com u can have connectivity
```
