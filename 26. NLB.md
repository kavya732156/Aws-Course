***Network Load Balancer (NLB)***:is a high-performance load balancer in Amazon Web Services (AWS) that operates at Layer 4 (Transport Layer) of the OSI model.<br>
It is designed to handle millions of requests per second with ultra-low latency.<br>

***Key Features of NLB***
1. Operates at Layer 4 : Works with TCP, UDP, and TLS protocols and Routes traffic based on IP address and port number.
2. High Performance : Handles extreme workloads, Maintains very low latency and Scales automatically.
3. Static IP Support : You can assign Elastic IPs to NLB and Useful when clients require fixed IP addresses.
4. Preserves Source IP : Backend servers can see the original client IP address and Useful for logging and security.
5. Cross-Zone Load Balancing : Distributes traffic across multiple Availability Zones. <br>

***Basic Architecture***<br>
Client → NLB → Target Group → EC2 Instances <br>
Targets can be:EC2 Instances, IP addresses, Containers (ECS) and On-prem servers (via IP mode) <br>

***Task***<br>
***1. Launch 2 EC2 Instances (Different AZs)***
```
Step 1: Launch Instance – Server 1
Go to EC2 → Launch Instance
Name: NLB-Server-1, AMI: Ubuntu Server 22.04 LTS, Instance Type: t2.micro (Free tier), Key Pair: Select existing or create new
Network: Same VPC, Choose Availability Zone 1 (Example: ap-south-1a)
Security Group: Allow SSH (22) from your IP, Allow HTTP (80) from Anywhere, Allow Custom TCP (80) from Anywhere (for NLB) and Launch Instance.

Step 2: Launch Instance – Server 2
Repeat same steps but: Name: NLB-Server-2 and Choose Different Availability Zone (Example: ap-south-1b)
```

***4. Create Target Group (For NLB)***
```
Go to EC2 → Target Groups, Click Create Target Group
Configuration: Target Type: Instances
Protocol: TCP, Port: 80
VPC: Select same VPC and Click Next.
Register Targets: Select Server-1 and Select Server-2
Click Include as pending below
Create Target Group
```

***5. Create Network Load Balancer (NLB)***
```
Go to EC2 → Load Balancers and Click Create Load Balancer
Choose Network Load Balancer
Basic Configuration: Name: My-NLB
Scheme: Internet-facing
IP type: IPv4
Listener: TCP → Port 80
VPC Mapping: Select VPC, Select both AZs, Select Public Subnets
Security Group: Select existing security group with: SSH (22), HTTP (80) and Custom TCP (80)
Listener & Routing: Protocol: TCP
Forward to: Existing Target Group (created earlier)
Click Create Load Balancer
```
