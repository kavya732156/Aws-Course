***Task 1 EBS Volume in Linux Server***
***1. Create an Instance By Using Linux Ami Use Instance Type “t2.micro”.***
   ~~~
   Go to EC2 → Instances → Launch Instance
   Choose Amazon Linux 2 / Amazon Linux 2023 AMI
   Instance type → t2.micro
   Select Key pair
   Security group → Allow SSH (22)
   Launch Instance
   ~~~
   <img width="1525" height="370" alt="image" src="https://github.com/user-attachments/assets/3ffa2a15-222d-4a07-ac15-d3db8cf15389" />
   
***2. Create one EBS Volume of Size 25GB.***
   ~~~
   Steps
   EC2 → Volumes
   Create volume
   Size: 25 GiB
   Volume type: gp3
   AZ: Same AZ as EC2 instance ⚠️ (Important)
   Create
   ~~~
   <img width="1560" height="333" alt="image" src="https://github.com/user-attachments/assets/2e8330ae-bafc-4bd3-a029-54017ac0fc6e" />
   
***3. Attach the Volume to EC2 instance***
   ~~~
   EC2 → Volumes
   Select your volume
   Actions → Attach volume
   Choose your EC2 instance
   Device: /dev/xvdbf
   Attach : Volume attached
   ~~~
   <img width="1909" height="763" alt="image" src="https://github.com/user-attachments/assets/7c016d21-f686-4282-9c3d-88822ceef64b" />

***4. Mount EBS Volume To Linux Instance.***
   ~~~
   sudo mkfs -t ext4 /dev/xvdbf
   sudo mkdir /mnt/kavya-ebs
   sudo mount /dev/xvdbf /mnt/kavya-ebs
   df -h
   ~~~

```

Assign Drive Letter → K:
```
